Project FleetCom

Project & Engineering Lead

About

During my senior year at Clark University, I was provided me with the opportunity to direct an eleven man team. Together we developed a unique sci-fi adventure that takes the player on a fast paced journey through a vast and dangerous world.

AI

If it weren't obvious already, traditional pathfinding isn't going to work in this game. Given the full 3 dimentions that need to be fully traversable, using Unreal Engine's built in pathing solutions aren't going to work, and traditional systems like A* simply wouldn't cut it. Sure, for the most part ship agents could just point and move at an enemy or follow a spline path and have ok results.

While we do use these methods at times, there are other times where the environment is too complex, or the movement becomes too rigid to have meaningful and fun interactions. To solve this, flight agents use a combination of spline paths, pathfinding, point to point movement and reactive manuevers to create engaging combat scenarios.

Firstly the elephant in the room, pathfinding. This was actually a very fun one to solve. In a sentence, I implemented a bi-directional rapidly exploring random tree. Given a starting point (ai agent) and an end point (agent goal), a bounding box is created around the points. Then the algorithm takes turns exploring random directions and distances from the start and end point, biasing them to head towards each other. If it discovers a shorter way to a point, it will rewire the exploration node to find the shortest possible path. Once both exploration branches have a point that can see a point on the other with an appropriate amount of space for the agents body to fit, the path is completed and returned as a list of points. This was a very enjoyable excersize. This was the first time, in terms of pathfinding, that I had to fully research and develop an appropriate system.

Map

After some discussion on how we wanted to present the map to the player, we decided to take inspiration from Legend of Zelda: BoTW's divine beast map and have a 3D holographic display that the player can fully explore.

I created the map model by making a simplified, scaled down model of the world in Blender. When a ship spawns in, it also spawns in a model that represents it, assigning it a material based on it's type (player, ally, nuetral, enemy) on a layer only the map camera can render, and updates its position when its attached ship moves.
